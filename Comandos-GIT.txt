git init - Indicar que essa pasta será o repositório

git ls-files - Verificar quais os arquivos que pertencem ao repositório.

git status - Verificar o estado dos arquivos do projeto.Se o arquivo não estiver sendo controlado, aparecerá com a cor vermelha.

git add arquivo.html - adicionar arquivos no repositório para o GIT controlar.

git add * - Adiciona todos arquivos do diretório para o GIT controlar.

- Informar ao Git o nosso nome e e-mail.Para identificação.Com isso o usuário será o responsável pelas alterações no repositório atual.
git config --global user.name "Lorran"
git config --global user.email "lorran@gmail.com"
- Caso essa configuração não seja efetuada, o Git vai determinar o nome de usuário atual do terminal como o autor das alterações.


git commit -m "Início do projeto" - Após incluir os arquivos é hora de comitar os arquivos para o GitHub.
- A flag "-m", indica que o conteúdo a seguir é a mensagem que será utilizada para descrever o que está sendo feito no commit.

- Adicionar repositório remoto.
git remote add [alias_do_repositorio] [uri_do_repositorio]
Exemplo:
https://github.com/[nome_do_usuario]/[nome_do_repositorio].git

git remote add https://github.com/brunosilvaJava/curso-git

- Comando completo, sendo que origin significa um alias

git remote add origin https://github.com/brunosilvaJava/curso-git

- Remover um servidor remoto
git remote remove origin

-- Comando git push, empurra para o servidor.Este comando é para copiar o projeto local para o servidor.

git push origin Dev

git log -- Visualizaçao de Log dos commits

git branch nome_branch - Para criar uma branch

git checkout nome_branch   - Para mudar de Branch

git pull origin nome_branch - Atualizar da branch remoto para local.(Atualização de repositório remoto to local)

git push origin nome_branch - Atualizar branch local pra remota.(Atualização de repositório local para remoto da repectiva branch)

git branch -r  - Visualizaçao branch remota
